  return (
    <>
      <BrowserRouter>
        <div className="min-h-screen flex flex-col">

            <Route path="/sobre" element={<Sobre />} />

          <Footer />
        </div>
      </BrowserRouter>
    </>
  );
}

export default App;

- Dinâmicas
na url, normalmente temos muitas coisas escritas e com &, que é uma concatenação. mas, se tiver "?", voce envia parametros.
exemplo: ?id=03903&0403
ou seja, sao caminhos que mudam com valores passados na URL. esses valores sao parametros, e permitem renderizar o mesmo componente com dados diferentes.

=> cuidado com o tipo de informação que voce envia pela url.
=> nao enviar login e senha pela url (informaçoes sensiveis).
=> ideal para paginas de produtos, usuarios, postagens, etc.

é necessário criar uma pasta e um arquivo, e criar a função. porém, adicionar uma outra const dentro, assim:

import { useParams } from "react-router-dom";

const ProdutoDetalhe = () => {
    const {id} = useParams();
  return (
    <>
      <main className="flex-1 container mx-auto p-6">
        <div className="grid grid-cols-1 gap-6">
          <h2 className="text-2xl font-semibold mb-2">Produto {id}</h2>
        </div>
      </main>
    </>
  );
};
export default ProdutoDetalhe;

O que é o useParams?
Imagine que você tem um site de e-commerce e as URLs dos produtos são:

meusite.com/produto/1
meusite.com/produto/2
meusite.com/produto/bola-de-futebol

O número 1, 2 ou o texto bola-de-futebol são parâmetros variáveis. O useParams serve justamente para o seu código React conseguir pegar esse valor da barra de endereço e usar dentro do componente.

const {id} = useParams();
useParams(): Essa função vai até a URL e puxa todos os parâmetros que foram definidos na sua rota.
{ id }: Isso é a desestruturação. O useParams devolve um objeto. Se na sua rota você definiu o nome como :id, você usa { id } para extrair esse valor direto para uma variável.
Resultado: A variável id agora guarda o valor que aparece lá na URL.

e para aparecer nas rotas, colocar:

<Route path="/produto/:id" element={<ProdutoDetalhe />} />
=> precisa colocar o :id porque foi o que declaramos.

- Na prática (O que o usuário vê)
Se o usuário acessar localhost:3000/produto/42:

O useParams() vai ler a URL.
Ele vai encontrar o valor 42.
A sua variável id passará a valer 42.
Na tela, o seu <h2> vai renderizar: Produto 42.

o que siginifica:
( {children} )
- É aqui que a mágica acontece dentro do HTML (JSX).
- O {children} entre chaves diz ao React: "O que quer que o usuário coloque dentro da tag <Card>, renderize aqui dentro desta div".

---------------------------------------

type CardProps = {
    children: ReactNode;
}
- type CardProps: Você está criando um "contrato" chamado CardProps.
- children: ReactNode: Você está dizendo: "Olha, esse componente vai receber algo chamado children, e o tipo dele é ReactNode".
- ReactNode: É um tipo bem genérico que aceita quase tudo: um texto, um botão, outro componente ou até uma lista de elementos.

-----------------------------------------

const Card = ({ children } : CardProps) => {
    return(
        <>
        <div className="bg-pink-100 p-4 rounded">
            {children}
        </div>
        </>
    )
}
export default Card;
- Aqui é a criação do componente propriamente dita.
- ({ children }): Isso se chama desestruturação. Em vez de escrever (props) e depois usar props.children, você já "pega" o children direto de dentro das propriedades que o componente recebe.
- : CardProps: Você está avisando ao React: "Este componente deve seguir as regras que eu criei ali em cima no type CardProps".

Basicamente, você está dizendo para o React: "Para cada item que existir na minha lista de produtos, crie um Card na tela."

Aqui está o "quem é quem" nesse código:

1. produtos.map((produto) => ( ... ))
O .map() é como um carimbo. Ele percorre o seu array de produtos.
Para cada item lá dentro, ele executa o que está entre os parênteses ( ).
A variável produto (no singular) representa o item atual que o carimbo está processando naquele momento.

2. <Card key={produto.id}>
O Componente: Você está usando aquele componente Card que criou antes.
O "Children": Tudo o que está dentro do <Card> e </Card> (o h2, o p e o Link) entra naquelas chaves {children}.
A key: É o RG do item. O React precisa disso para não se perder se você deletar ou mover um produto da lista.

3. As Chaves { } e Template Strings ` `
No React, sempre que você quer colocar código JavaScript (como variáveis) dentro do HTML, você usa chaves.

Produto {produto.id}: Se o ID for 10, na tela vai aparecer "Produto 10".
to={/produto/${produto.id}} : Isso cria o link dinâmico.

Se for o produto 1, o link vira /produto/1.
Se for o produto 5, vira /produto/5.
