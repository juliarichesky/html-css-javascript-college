useEffect

- hook que permite lidar com efeitos colateris em componentes funcionais.
- efeitos colaterais sao opracoes que afetam algo fora do escopo de função, como modificar DOM, buscar dados de uma API, configurar um time ou adicionar event listeners.

import { useEffect } from "react";
    useEffect(() => {  
        // Código a ser executado no efeito
    }
);

QUANDO USAR?
- quando precisar realizar operações assíncronas, como chamadas de API, manipulação 
do DOM, gerenciamento de assinaturas de eventos ou qualquer operação que não seja parte do fluxo de renderização principal.
- quando você precisa realizar efeitos colaterais, como modificar o DOM, fazer requisições  HTTP, lidar com eventos globais (resize, keydown etc.) ou interagir com APIs externas. ele roda após a renderização e pode ser controlado por dependências.
- chamadas de API → Buscar dados e atualizar o estado
- modificar o DOM fora do JSX → Exemplo: alterar document.title
- event listeners globais → resize, scroll, keydown, etc.
- timers e intervalos → setTimeout, setInterval
- subscrições → WebSockets, Firebase, etc.
- sincronizar estado com localStorage ou sessionStorage.


QUANDO NÃO USAR?
- para exibir ou atualizar o JSX com estados ou props (o React já faz isso automaticamente)
- para calcular valores derivados do estado (use useMemo ou diretamente no JSX)

ACESSAR API

para buscar dados de uma API (como o JSONPlaceholder), usamos o fetch dentro do useEffect. isso evita que a requisição seja feita infinitas vezes a cada renderização.

1. estrutura

useEffect(() => {
  fetch("https://jsonplaceholder.typicode.com/users") // 1. URL da API
    .then((response) => response.json())            // 2. converte a resposta para JSON
    .then((data) => setUsers(data));                // 3. guarda os dados no estado
}, []); // 4. array vazio = executa apenas UMA VEZ ao carregar a página


2. como o react/typeScript não sabe o que vem da API, precisamos "avisar" qual o formato dos dados para não dar erro no código:

// definimos o molde do objeto que vamos receber
type User = {
  id: number;
  name: string;
  address: {
    street: string;
  };
};

// no useState, indicamos que ele receberá um array de Usuários:

const [users, setUsers] = useState<User[]>([]);


3. após buscar os dados da API e guardá-los no estado users, precisamos exibi-los na tela. para isso, usamos o método .map() e a lógica de renderização condicional.


  <h2>Lista de Usuários</h2>
    <ul>
      {users.map((user) => (
        <li key={user.id}> 
          {user.name}
          <ul>
            <li>Email: {user.email}</li>
            <li>Rua: {user.address.street}</li>
            <li>Cidade: {user.address.city}</li>
          </ul>
        </li>
      ))}
    </ul>

O ARRAY DE DEPENDÊNCIAS [ ]
o useEffect aceita um segundo argumento (o array []), que controla quando o efeito deve rodar:

useEffect(() => { ... }) (sem array): roda em toda renderização. (cuidado: pode causar loops infinitos).

useEffect(() => { ... }, []) (array vazio): roda apenas uma vez, quando o componente nasce (montagem). Ideal para chamadas de API.

useEffect(() => { ... }, [cont]) (com variável): roda toda vez que o valor de cont mudar.

PONTOS IMPORTANTES:

users.length === 0: Verifica se o array ainda está vazio (estado inicial). se estiver, mostra a mensagem de "Carregando".

.map((user) => ...): percorre cada objeto dentro do array users e transforma cada um em um item de lista (<li>).

key={user.id}: obrigatório! o react precisa dessa chave única para identificar cada item. se você esquecer, ele dará um aviso no console.
